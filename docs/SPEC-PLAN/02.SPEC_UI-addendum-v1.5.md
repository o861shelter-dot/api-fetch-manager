## 1. Mục tiêu
Một toggle **UI Self-Test Mode**. Khi bật: hệ thống **tự điền dữ liệu giả → tự click mọi thao tác → ghi log toàn bộ → tự kiểm bằng quy ước dữ liệu → show kết quả từng chức năng** (kèm text của element trên form). Trọng tâm: **mọi tính năng liên quan tới fetch đều phải có kết quả đầu ra kiểm chứng được.**

## 2. Cơ chế
- Toggle ở Topbar (icon 🧪 + tooltip) hoặc hotkey. Chạy trên **owner sandbox + dữ liệu GIẢ**, KHÔNG ghi RTDB thật (dùng storage memory/file cho test run).
- **Test runner** tuần tự mỗi màn: (1) auto-fill field bằng fixture → (2) auto-click action → (3) capture snapshot (giá trị + text element) → (4) assert theo quy ước → (5) ghi kết quả.
- **Log toàn bộ** vào `rtdb-logs` scope `self-test` (credential vẫn masked): mỗi bước lưu `elementSelector`, `elementText`, `valueBefore`, `valueAfter`, `assertion`, `pass/fail`.
- **Panel kết quả**: danh sách chức năng → ✓/✗ + text element liên quan + diff kỳ vọng vs thực tế. Dùng `DataList` (filter/sort/export JSON/CSV/PDF).

## 3. Assertion cho nhóm Fetch (BẮT BUỘC có kết quả đầu ra)
- **Save Fetch Flow**: capture giá trị từ el nào (name/url/headers/body/extract) → build curl thế nào → so khớp curl với input.
- **Placeholder resolve**: nếu có `{{placeholder}}` → assert url fetch thực tế **đã chứa giá trị resolved**, không còn `{{...}}`; credential vẫn masked.
- **Test step**: assert response parse được + JSONPath extract đúng field.
- **Execute flow**: assert từng step có `responseStatus`, `durationMs`, `extract` vào context; `pinToVar` ghi đúng variables.
- **History/Logs**: assert mỗi fetch tạo đúng bản ghi history + log.

**Ví dụ output (case Save Fetch Flow):**
```json
{
  "feature": "save-fetch-flow",
  "captured": {
    "el.name": "GitHub - Tạo repo",
    "el.url": "https://api.github.com/user/repos",
    "el.header.Authorization": "Bearer {{github.token}}",
    "el.body": "{ \"name\": \"{{repoName}}\" }"
  },
  "builtCurl": "curl -X POST 'https://api.github.com/user/repos' -H 'Authorization: Bearer ***' -d '{ \"name\": \"demo\" }'",
  "assertions": [
    { "name": "curl chứa method POST", "pass": true },
    { "name": "placeholder {{repoName}} resolve = demo", "pass": true },
    { "name": "url fetch không còn {{...}}", "pass": true },
    { "name": "credential masked trong log", "pass": true }
  ],
  "result": "pass"
}
```

## 5. API hỗ trợ
| Method | Endpoint | Mô tả |
| --- | --- | --- |
| POST | `/api/selftest/run` | Body `{ scope?, scenarioIds? }` → chạy self-test (dry-run executor với fixture), trả kết quả tổng hợp |
