# API Fetch Manager — SPEC\_UI (Design System / Skill)
> **Đây là tài liệu SKILL về giao diện.** Bất kỳ agent nào code UI cho dự án PHẢI tuân thủ 100% tài liệu này. Mọi con số là bắt buộc, không phải gợi ý. Khi mâu thuẫn, tài liệu này là nguồn chân lý cho UI.
**Version:** 1.0 · **Ngày:** 2026-07-07 · **Nền tảng:** CSS variables thuần (không phụ thuộc UI framework nặng)

* * *
## 0\. Nguyên tắc bất biến (Golden Rules)
1. **Responsive** desktop + mobile, thống nhất toàn hệ thống.
2. **2 theme**: sáng (light) + tối (dark). Mọi màu qua CSS variable, không hardcode.
3. **Font mảnh**: font-weight 300–400. **Khoảng cách gần**: spacing scale 4px, giao diện dày đặc.
4. **KHÔNG dùng** **`alert()`** **/** **`confirm()`** **/** **`prompt()`** **của browser.** Mọi thông báo qua **modal form**.
5. Mọi form/modal **tự detect chiều cao & rộng** → tự tạo scrollbar khi tràn.
6. Modal **bắt buộc có nút thoát (X)**; **click ra ngoài KHÔNG đóng**; ESC đóng được (tùy chọn).
7. Mọi **button có icon + tooltip** mô tả chức năng và kết quả.
8. **Chức năng quan trọng** (xóa, execute API, rotate key, import ghi đè) phải có **modal confirm**.

## 6\. Inspect Element Mode — UI spec
**Toggle bật/tắt** ở Topbar (icon 🎯 + tooltip "Bật chế độ tạo bug/issue từ giao diện").

Khi bật:

1. Con trỏ đổi thành crosshair. Overlay `z-index: var(--z-inspect)`.
2. Hover element → viền highlight (outline 2px `--primary`, nền `--primary` alpha .08).
3. Click → chọn element (thêm class `.inspect-selected`, viền `--success`). Click lại → bỏ chọn. **Chọn được nhiều element.**
4. Nút nổi "Tạo issue (n)" hiện số element đã chọn → mở **modal form** (title, description, expectedResult) + preview danh sách element (selector rút gọn).
5. Lưu → modal notify success. Danh sách element kèm `selector`, `outerHTML`, `boundingRect` ghi vào issue.
6. Có màn quản lý issue: table/card + filter theo type/status, sửa/xóa qua modal, nút Export/Copy Markdown.

```css
.inspect-hover { outline: 2px solid var(--primary); outline-offset: 1px;
  background: color-mix(in srgb, var(--primary) 8%, transparent); }
.inspect-selected { outline: 2px solid var(--success); }
```

## 10\. Bổ sung v1.1 — UI cho Fetch Flow, Variables & Placeholder Engine
Toàn bộ component dưới đây **kế thừa design tokens, modal rules, tooltip, scrollbar** ở mục 1–9. Không tạo pattern mới.
### 10.1 Fetch Builder — chế độ Flow (multi-step)
Layout 2 pane trong modal builder (full-height, tự scroll):
*   **Pane trái — Steps list**: danh sách step dạng vertical, kéo-thả sắp xếp. Mỗi step card hiện method badge + tên + nút xóa (icon + tooltip). Nút "➕ Thêm step".
*   **Pane phải — Step editor**: form cấu hình step đang chọn.

```plain
┌── Fetch Flow Builder ─────────────────────────────┐
│ [Steps]            │  [Step editor: auth]          │
│  ┌─────────────┐   │  Method [POST▾]  URL [____]   │
│  │ 1. auth  ✕  │   │  Headers (key/value rows)     │
│  │ 2. createRepo│  │  Body (textarea, mono)        │
│  └─────────────┘   │  Extract: field + JSONPath[+] │
│  [+ Thêm step]     │  [ ] pin vào biến {{var.__}}  │
└────────────────────┴───────────────────────────────┘
```

**Step editor gồm:**
*   Method (select) + URL template (input, hỗ trợ placeholder highlight).
*   Headers: các dòng key/value (nút thêm/xóa dòng, icon + tooltip).
*   Body template: textarea mono, `--fs-sm`, `--fw-thin`.
*   **Extract block**: mỗi dòng = `field name` + `JSONPath` + toggle "pin vào biến" (chọn/nhập varKey). Nút ➕ thêm dòng.
*   **Test API theo step**: nút "Test API" ở step đang chọn, nhận params runtime dạng JSON, gọi API thật và hiển thị toàn bộ response; JSON được format đẹp để user chọn JSONPath extract. Có nút "Copy curl" xuất lệnh curl từ method/url/header/body của step và params hiện tại.
*   Placeholder trong input/textarea được **highlight** (nền `--primary` alpha .08) và có autocomplete gợi ý `{{ctx.*}}`, `{{var.*}}`, `{{input.*}}`, credential keys.
### 10.2 Inputs config (source runtime/store/context)
Trong builder, mỗi input là 1 row:
`[tên input] [source ▾: runtime | store | context] [required ☑] [ref: varKey / stepId.field]`
*   Nếu `source=runtime` → khi execute sẽ render form hỏi (mục 10.3).
*   Icon + tooltip giải thích từng source.
### 10.3 Execute Modal (input runtime)
Khi execute flow có input `runtime` → **modal form** (chuẩn mục 4.2) hỏi lần lượt các input runtime trước khi chạy. Modal có nút X, click ngoài không đóng, tự scroll nếu nhiều field. Nút "Chạy" là **chức năng quan trọng** → hiện confirm nếu flow có side-effect (tạo repo, tạo DNS...).

**Trong lúc chạy:** hiển thị **step progress** (stepper dọc: pending → running(spinner) → success ✓ / error ✗). Step lỗi mở được panel log chi tiết.
### 10.4 View "Extracted Data"
Mục nav riêng ở sidebar (icon 🗃 + tooltip "Dữ liệu trích xuất từ các lần fetch").
*   Filter: emailOwner, service, template.
*   Desktop table / mobile card: cột **Field**, **Value** (vd repo name, git link, cloudflare NS id), **Template nguồn**, **Thời điểm**.
*   Mỗi dòng có nút 📌 "Pin thành biến" (icon + tooltip) → confirm modal → ghi vào kho biến.
*   Value dài → truncate + tooltip/expand; giá trị nhạy cảm vẫn masked như credential.
### 10.5 View "Variables" (kho placeholder)
Mục nav riêng (icon 🔤 + tooltip "Kho biến tái sử dụng").
*   2 tab: **Global** và **Theo owner**.
*   Table/card: `key`, `value`, `source (manual/extracted)`, `updatedAt`.
*   CRUD qua modal (thêm/sửa/xóa). Xóa = chức năng quan trọng → confirm modal.
*   Nút copy `{{var.key}}` (icon + tooltip "Copy tham chiếu để dán vào fetch").
### 10.6 Placeholder Engine — Editor UI
Ở bất kỳ ô nhập có placeholder (URL, header, body):
*   **Autocomplete dropdown** khi gõ `{{` → gợi ý nguồn (`ctx`, `var`, `input`, credential) + danh sách transform sau dấu `|`.
*   **Transform pipe builder** (tùy chọn): UI chip cho phép chọn transform (`upper`, `lower`, `trim`, `replace`, `base64`, `date`...) thay vì gõ tay.
*   **Advanced JS editor**: mở qua modal riêng, textarea mono có syntax hint, kèm badge cảnh báo "Chạy trong sandbox: cấm network/fs, timeout 200ms". Nút "Test" chạy thử với giá trị mẫu → hiện kết quả trong modal, không dùng alert.
