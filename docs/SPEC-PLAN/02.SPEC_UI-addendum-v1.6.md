## 1. Docs viewer KHÔNG che UI
- **Không dùng modal full-screen** cho docs. Dùng **side panel/drawer** trượt phải (resizable, ghim được) hoặc split-pane cạnh builder → vừa đọc docs vừa thao tác form.
- Nguồn: render **trực tiếp markdown** từ `docs/services/<host>.md`. Có ô search trong panel + mục lục anchor theo heading.
- Có **link external** mở tài liệu gốc nhà cung cấp để xem đầy đủ.

## 2. Tích hợp Fetch Builder
- Nút "📖 Docs" trong builder mở panel docs của **đúng service đang chọn**.
- Mỗi mẫu curl trong docs có nút **"Dùng mẫu này"** → nạp thẳng vào builder (parse-curl).

## 3. Agent BẮT BUỘC fetch & biên soạn tài liệu API
Fetch tài liệu API thật cho mọi dịch vụ trong credentials mẫu (`00.YEUCAU.md` §6): **github.com · cloudflare · dpdns · tailscale.com · supabase.com · cron-job.org**. Mỗi `docs/services/<host>.md` bố cục gần giống trang API nhà cung cấp:
- Tổng quan + base URL + auth (map tới credential key nào).
- Từng endpoint/nghiệp vụ: **mục đích (tác dụng lệnh)** · **curl mẫu đầy đủ** (placeholder credential, KHÔNG secret thật) · tham số · **response mẫu đầy đủ (JSON)** · field đáng trích (gợi ý JSONPath).
- **Link external** tới doc gốc của endpoint.
- Nhóm nghiệp vụ đúng thực tế (GitHub: repos/actions/secrets/runners · Cloudflare: zones/dns/tunnels · cron-job.org: jobs/history…).
