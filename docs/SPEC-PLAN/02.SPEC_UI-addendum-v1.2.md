# SPEC_UI — Addendum v1.2 (rule xuyên suốt toàn dự án)

> Bổ sung cho `02.SPEC_UI.md`. Các rule dưới đây **áp dụng toàn hệ thống**, mọi trang/component phải tuân theo. Khi mâu thuẫn, addendum này ưu tiên cho các mục nó đề cập.

**Ngày:** 2026-07-08 · **Áp dụng từ:** batch cải tiến Inspect + Fetch Builder + UI chung.

---

## 1. Tooltip tự định vị (thay pseudo `::after`)
- Tooltip render bởi `frontend/src/lib/tooltip.ts` (global), `position: fixed`, **tự flip trên/dưới** khi chạm mép, **clamp trong viewport**, cho phép wrap (`white-space: normal`).
- **CẤM** dùng lại pseudo `[data-tooltip]:hover::after` (gây cắt chữ ở mép Topbar).
- Mọi tooltip vẫn khai qua thuộc tính `data-tooltip="..."` — không đổi API.

## 2. Rule khoảng cách label ↔ value
- Mọi cặp nhãn/giá trị dùng class `.kv-row` (hoặc `.card__row`) với `gap: var(--sp-2)` trở lên. **CẤM** đặt label và value sát nhau không giãn cách.
- Áp dụng: Credentials, History, Extractions, Variables, Issues, mọi card/detail.

## 3. Combobox (nhập tự do + chọn danh mục)
- Component chung `Combobox`: cho gõ tay HOẬC chọn từ danh mục; có nút "Lưu vào danh mục".
- Danh mục lưu **chung toàn workspace** (không theo owner) — backend `/api/catalogs?field=`.
- Dùng cho: Tên flow / Service / Business trên Fetch Builder (và form tương lai).

## 4. KeyPicker + placeholder inserter
- Component chung `KeyPicker`: chọn credential key của owner → chèn `{{key}}` vào input đang focus.
- Kèm ô advanced JS (sandbox, hàm sẵn) áp lên giá trị trước khi chèn; nút **xem giá trị resolved** (gọi `/api/engine/resolve`).
- Dùng cho mọi input template có placeholder.

## 5. Inspect Mode (bổ sung)
- **Hotkey toàn cục** bắt đầu inspect: mặc định **Ctrl+Shift+J**, override qua env `VITE_API_FETCH_MANAGER_INSPECT_HOTKEY`. Hoạt động ở **bất kỳ form/chức năng/modal** nào.
- Thanh công cụ inspect có 3 nút: **Tạo issue (n)**, **Tạm ngưng** (giữ element đã chọn, ngừng bắt sự kiện để thao tác form khác), **Thoát**. **ESC = thoát**.
- Khi mở form tạo issue hoặc tạm ngưng → **nhả con trỏ** (body thêm class `inspect-paused`), không bắt click nền.
- Inspect **phải chạy được trên modal** (overlay khi inspect nằm dưới z-inspect; chỉ loại trừ thanh công cụ inspect).
- Mỗi element chọn lưu thêm **text** (innerText cắt gọn). Panel "Element đã chọn" hiển thị **mỗi item 1 row có số thứ tự**, 2 cột **Selector** + **Text đã chọn**. Issue + markdown export kèm text.

## 6. Modal cấu hình Admin token
- Input **canh giữa**, label **trên** input, rộng cân đối (dùng `.field--center`, `.modal--narrow`).

## 7. Guard thiếu Admin API token
- Khi backend trả **401** → hiển thị thông báo chuẩn: "Chưa cấu hình Admin API token, hãy lưu trước khi sử dụng" (không phải lỗi fail kỹ thuật). Đã xử lý ở `api.ts` (`ADMIN_TOKEN_REQUIRED_MSG`).

## 8. Fetch Flow Builder (bổ sung)
- Tên flow / Service / Business = Combobox (§3), có lưu danh mục dùng chung nhiều owner.
- Header/giá trị có preset tái dùng (Authorization, Content-Type...); chèn key credential qua KeyPicker (§4); xem resolved.
- Body template chọn định dạng (JSON | raw | form...), có nút **Beautify**.
- **Tạo từ mẫu / Lưu thành mẫu**: flow presets lưu trong **database** (node `flow-presets`, cờ `isPreset`), lấy ra dùng cho owner khác. Seed sẵn 4 mẫu: fetch repo GitHub, get user GitHub, dispatch workflow GitHub, lấy account id Cloudflare.

## 9. Cập nhật checklist tuân thủ
- [ ] Tooltip không bị cắt chữ ở mép (3 breakpoint)
- [ ] Không còn cặp label/value dính nhau
- [ ] 401 → thông báo "chưa cấu hình Admin API token"
- [ ] Credentials không hiển thị trùng row + card
- [ ] Hotkey Ctrl+Shift+J bật inspect mọi nơi; ESC thoát; có nút tạm ngưng/thoát
- [ ] Inspect chạy trên modal; nhả con trỏ khi tạo issue
- [ ] Panel element có số thứ tự + selector + text
- [ ] Fetch Builder: combobox danh mục, KeyPicker, body beautify, tạo/lưu mẫu
