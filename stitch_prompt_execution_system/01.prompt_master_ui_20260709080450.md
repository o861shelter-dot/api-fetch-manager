## 2\. Golden Rules (BẤT BIẾN — vi phạm = reject)
1. **Responsive** desktop + mobile, cùng 1 bộ component.
2. **2 theme** sáng/tối, mọi màu qua CSS variable.
3. **Font mảnh 300–400**, spacing 4px, giao diện dày đặc.
4. **KHÔNG dùng** **`alert/confirm/prompt`** **của browser.** Mọi thông báo qua modal (`ui.notify`/`ui.confirm`).
5. Mọi form/modal **tự detect chiều cao/rộng → tự scrollbar** khi tràn (`max-height:90vh; overflow:auto`).
6. Modal **có nút ✕**; **click ngoài KHÔNG đóng**; ESC đóng; trap focus.
7. Mọi **button có icon + tooltip** (mô tả chức năng + kết quả, không chỉ lặp label).
8. **Chức năng quan trọng** (xóa, execute, reveal, import ghi đè, xóa cron…) phải có **confirm modal**.
9. Tooltip **tự flip/clamp trong viewport** (không dùng pseudo `::after` gây cắt chữ ở mép).
10. **\[RULE\] Mọi danh sách** (table/card) BẮT BUỘC có **filter · sort · export data (JSON/CSV) · export PDF** — dùng chung component `DataList`.
11. **\[RULE\] Docs dịch vụ hiển thị KHÔNG che UI** đang thao tác (side panel/split-pane, không modal full-screen).

## 3\. Component nền tảng (dựng trước)
*   **Button** `.btn` (h 30px, icon + text, `data-tooltip` bắt buộc): biến thể `--primary` (nền emerald, chữ near-black), `--danger`, `--ghost`. `:disabled` opacity .5.
*   **Modal**: overlay + `.modal` (width `min(560px,100%)`, `max-height:90vh`, flex column). Head (title + ✕) · body (overflow auto) · foot (actions phải). Click overlay KHÔNG đóng; ESC đóng; trap focus.
*   **NotificationModal** (thay alert): success/error/info/warning khác icon + màu accent.
*   **ConfirmModal**: dùng cho mọi chức năng quan trọng.
*   **Field / Input / Select / Textarea**: label trên, `--fw-thin`, focus outline 2px `--primary`.
*   **Tooltip**: global (`position:fixed`, tự flip/clamp, wrap được), khai qua `data-tooltip`.
*   **Table/Card list**: desktop row 32px font 12px; <768 → card. Giá trị nhạy cảm luôn **masked** + nút 👁 (confirm trước khi lộ).
*   **Scrollbar** mảnh đồng bộ theme. **Icon set** Lucide, stroke 1.5, nhất quán.
*   **DataList** (component chung, dùng cho MỌI danh sách): props `columns`, `rows`, `filter`, `sort` (đa cột asc/desc), toolbar export **JSON · CSV · PDF** (mỗi nút icon+tooltip; PDF giữ theme + tiêu đề + thời điểm + owner context). Export PDF dùng lib nhẹ client-side.

## 6\. Các trang cần dựng
### 6.1 Owners
*   `DataList`: email · isSaveRtdbEmail · createdAt · số credential.
*   CRUD qua modal (xóa = confirm, cảnh báo kèm credentials).
*   Mỗi dòng có cụm action (icon+tooltip): **Active lên Topbar · Tạo Fetch Builder · Xem credentials · Đi tới dịch vụ**.
### 6.2 Credentials (theo owner)
*   `DataList`: key · label · **credId (rút gọn)** · service · value **masked** + 👁 (confirm reveal).
*   CRUD; 1 key có thể có **nhiều giá trị** (nhiều credId) → hiển thị phân biệt bằng label + credId. (Chi tiết credId: §8.)
### 6.3 Fetch Builder / Flow (multi-step) — trọng tâm
*   Layout 2 pane trong builder: **trái** = steps list (kéo-thả, method badge + tên + xóa, nút ➕); **phải** = step editor.
*   Step editor: Method (select) · URL template · Headers (rows key/value) · Body (textarea mono, nút **Beautify**) · **Extract block** (field + JSONPath + toggle pin var).
*   **Placeholder highlight** (nền `--primary` alpha .08) + **autocomplete** khi gõ `{{` (gợi ý `ctx.* / var.* / input.*` + credential keys) + transform sau `|`.
*   **KeyPicker**: chọn credential của owner → chèn `{{key}}`. Nếu key có nhiều giá trị → **buộc chọn đúng bản ghi** (hiện label + masked + credId).
*   **Combobox** cho Tên flow / Service / Business (nhập tay hoặc chọn danh mục, có "Lưu vào danh mục").
*   **Test API theo step**: nhập params runtime (JSON), gọi thật, hiện response **format đẹp** để chọn JSONPath; nút **Copy curl** (giữ placeholder credential).
*   **Tạo từ mẫu / Lưu thành mẫu** (flow presets).
*   Nút **📖 Docs** mở panel tài liệu service đang chọn (§7).
### 6.4 Execute Modal
*   Flow có input `runtime` → modal hỏi lần lượt (chuẩn §3). Nút "Chạy" = quan trọng → confirm nếu có side-effect.
*   Khi chạy: **stepper dọc** (pending → running spinner → success ✓ / error ✗). Step lỗi mở panel log chi tiết.
### 6.5 History & Logs
*   `DataList` history (filter service/success) + logs (filter service/business/level). Che token.
### 6.6 Extracted Data
*   `DataList`: Field · Value (masked nếu nhạy cảm) · Template nguồn · Thời điểm. CRUD. Nút 📌 **Pin thành biến** (confirm).
### 6.7 Variables
*   2 tab: **Global** / **Theo owner**. `DataList`: key · value · source(manual/extracted) · updatedAt. CRUD qua modal. Nút copy `{{var.key}}`.
### 6.8 Services & Resources (dạng tab động)
*   **Mỗi dịch vụ = 1 tab** (GitHub · Cloudflare · Supabase · [cron-job.org](http://cron-job.org)…), thêm dịch vụ → thêm tab.
*   Trong tab: resource type + `DataList` item. **Từng item** có action: **Tạo Fetch Builder từ item** (bơm context item vào params) · **Lấy var** (extract field → pin). Ví dụ [cron-job.org](http://cron-job.org): mỗi job có nút **Gọi xóa theo id** (`{{item.jobId}}`, confirm).
*   Thêm/sửa/xóa service qua modal (host là Combobox).
### 6.9 Inspect Mode (giữ nguyên hành vi hiện có)
*   Toggle 🎯 + **hotkey Ctrl+Shift+J** (mọi nơi, kể cả trên modal). Hover highlight, chọn nhiều element (lưu selector + outerHTML + rect + **text**), tạo issue → export/copy Markdown. Thanh công cụ: **Tạo issue (n) · Tạm ngưng · Thoát** (ESC = thoát); khi mở form issue → **nhả con trỏ**.

## 7\. Docs viewer dịch vụ (không che UI)
*   **Side panel/drawer** trượt phải (resizable, ghim được) hoặc split-pane cạnh builder → vừa đọc vừa thao tác form.
*   Render **markdown trực tiếp** từ `docs/services/<host>.md`. Có ô search + mục lục (anchor theo heading) + **link external** tới doc gốc nhà cung cấp.
*   Trong builder: nút 📖 Docs mở đúng service; mỗi curl mẫu có nút **"Dùng mẫu này"** → nạp vào builder (parse-curl).

## 8\. Self-Test UI Mode (🧪)
*   Toggle ở Topbar. Bật → chạy trên **owner sandbox + dữ liệu giả**, KHÔNG chạm dữ liệu thật.
*   Runner tuần tự mỗi màn: auto-fill (fixture §9) → auto-click → capture (giá trị + **text element**) → assert theo quy ước → ghi kết quả.
*   **Panel kết quả** (dùng `DataList`, export được): mỗi chức năng ✓/✗ + text element + **diff kỳ vọng vs thực tế**.
*   **Bắt buộc với mọi tính năng fetch có output kiểm chứng:** case "Save Fetch Flow" phải show lấy giá trị el nào → build curl thế nào → placeholder resolve ra giá trị gì → **url thực tế đã chứa giá trị đó chưa** (không còn `{{...}}`); credential vẫn masked.

## 9\. Mock API để UI chạy độc lập
Vì làm UI riêng, dựng lớp `api.ts` trả **fixture** (có thể bật cờ `USE_MOCK`) cho: owners, credentials (nhiều key trùng có credId khác nhau), templates/flow-presets, extractions, variables, services/resources, history, logs, `/api/meta` (commit + build time). Response chuẩn `{ ok, data?, error? }`. Khi nhập lại hệ chính → tắt mock, dùng client thật (không đổi component).
> **Lưu ý credId (đồng bộ với backend):** mỗi credential có `credId` duy nhất tự sinh; placeholder map theo **credId** không phải key. Key nhiều giá trị → KeyPicker buộc chọn. UI phải lưu credId đã chọn vào `credentialRefs`, không lưu theo key.
